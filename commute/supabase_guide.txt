# Supabase Integration Guide for Developers

This document provides a guide for developers on how to use Supabase for authentication, credit deduction, storage, and other user management tasks within this application.

## 1. Authentication with Supabase through API

### Frontend (TypeScript)
The application uses `@supabase/supabase-js` for authentication.
- **Client Initialization**: Located in [`services/supabase.ts`](services/supabase.ts).
- **Authentication Context**: [`contexts/AuthContext.tsx`](contexts/AuthContext.tsx) provides `signInWithPassword`, `signOut`, and `user` object.
- **Login Implementation**: [`components/Login.tsx`](components/Login.tsx) handles the UI for signing in.

Example of signing in via API:
```typescript
import { supabase } from './services/supabase';

const { data, error } = await supabase.auth.signInWithPassword({
  email: 'user@example.com',
  password: 'password123',
});
```

### Backend (Python)
The backend validates the Supabase JWT token in [`backend/backend/api/deps.py`](backend/backend/api/deps.py).
- **get_current_user**: Extracts the Bearer token from the request, verifies it with `supabase.auth.get_user(token)`, and fetches or creates a user profile in the `profiles` table.

## 2. Credit Deduction through API

Credit deduction is handled via a PostgreSQL RPC function named `deduct_credits`.

### RPC Function Signature
The function `public.deduct_credits` is defined in [`backend/alembic/versions/e5f8a9b3c1f3_add_credit_system.py`](backend/alembic/versions/e5f8a9b3c1f3_add_credit_system.py).
- `p_user_id`: UUID of the user.
- `p_cost_amount`: Integer amount to deduct.
- `p_app_id`: String identifier for the app.
- `p_feature_name`: String identifier for the feature.
- `p_metadata`: JSONB object for additional context.

### Backend Implementation
The [`CreditService`](backend/backend/services/credit_service.py) in Python handles these calls using the **Service Role Key** to bypass Row Level Security (RLS).

```python
from backend.core.supabase import get_supabase_client

supabase = get_supabase_client(use_service_role=True)
response = supabase.rpc('deduct_credits', {
    'p_user_id': user_id,
    'p_cost_amount': cost,
    'p_app_id': app_id,
    'p_feature_name': feature,
    'p_metadata': metadata
}).execute()
```

## 3. Supabase Storage Usage through API

The current application primarily uses local storage in the `uploads/` directory for documents (see [`backend/backend/services/document_service.py`](backend/backend/services/document_service.py)). However, Supabase Storage can be integrated using the following patterns:

### Frontend
```typescript
const { data, error } = await supabase.storage
  .from('bucket-name')
  .upload('path/to/file', fileObject);
```

### Backend
```python
supabase = get_supabase_client(use_service_role=True)
with open('path/to/file', 'rb') as f:
    supabase.storage.from_('bucket-name').upload('path/in/storage', f.read())
```

## 4. Change User Password

### Through Frontend (Self-Service)
Implemented in [`components/Profile.tsx`](components/Profile.tsx). It first re-authenticates the user with their current password and then calls `updateUser`.

```typescript
// Update password for the currently logged-in user
const { data, error } = await supabase.auth.updateUser({
  password: 'new-secure-password'
});
```

### Through Developers (Admin/Service Role)
Developers can reset a user's password using the Supabase Dashboard or via the Admin API using the Service Role Key:

```python
# Using Python Admin API
supabase = get_supabase_client(use_service_role=True)
auth_admin = supabase.auth.admin
auth_admin.update_user_by_id(
    user_id,
    attributes={'password': 'new-password'}
)
```

## 5. List Usage History for Current User

Usage history is stored in the `public.usage_logs` table.

### Querying Usage History
To fetch history for the current user:

```typescript
const { data, error } = await supabase
  .from('usage_logs')
  .select('*')
  .eq('user_id', current_user_id)
  .order('consumed_at', { ascending: false });
```

The table structure:
- `id`: UUID (Primary Key)
- `user_id`: UUID (Foreign Key to auth.users)
- `app_id`: TEXT
- `feature_name`: TEXT
- `credits_consumed`: INTEGER
- `metadata`: JSONB
- `consumed_at`: TIMESTAMPTZ

## 6. Important Information for Developers

- **Service Role Key vs. Anon Key**: Use `get_supabase_client(use_service_role=True)` in the backend for admin operations (like credit deduction) that should bypass RLS. NEVER expose the Service Role Key in the frontend.
- **Profiles Table**: The `public.profiles` table is automatically populated upon the first login via [`backend/backend/api/deps.py`](backend/backend/api/deps.py). It stores `subscription_credits` and `topup_credits`.
- **Environment Variables**:
    - `SUPABASE_URL`: Your Supabase project URL.
    - `SUPABASE_KEY`: Anon key (safe for frontend).
    - `SUPABASE_SERVICE_ROLE_KEY`: Service role key (backend only).
- **RLS (Row Level Security)**: Ensure RLS is enabled on `profiles` and `usage_logs` tables. The `deduct_credits` function uses `security definer` to run with owner privileges, allowing it to update balances safely.
