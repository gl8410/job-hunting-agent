# Store and Retrieve File Embeddings from Qdrant

## 1. Overview
In this project, we utilize **Qdrant** as our vector database. To avoid system proxy interference (which often impacts Docker environments), we interact with Qdrant via its **REST HTTP API** through **requests (v2.32.5)** using `requests.Session()` with `trust_env=False`. We also have the **qdrant-client (v1.16.2)** available, but prioritize direct REST for critical docker proxy bypasses.

## 2. Preparing and Generating Embeddings
Embeddings are generated by chunking textual content and calling an external Embedding API (like SiliconFlow) in parallel batches to speed up the process.

1. **Text Extraction**: Read visual text via LLM vision models, parse PDF/DOCX using standard libs, or handle plain text.
2. **Chunking**: Use sliding window chunking to split large text (e.g., length=2000, overlap=200).
3. **Embedding Call**: Process chunks into embeddings array with concurrent threads to improve throughput.

## 3. Direct Qdrant REST API Interaction

> [!WARNING]  
> **FastAPI Async I/O Safety:** Because `requests` is a synchronous, blocking library, you **MUST NOT** use it directly inside `async def` FastAPI endpoints, as it will block the ASGI event loop. All `requests` calls to Qdrant must be wrapped in `asyncio.to_thread()`, or executed within a Background ThreadPool worker.

### Setup Client Session
Always configure `requests` to bypass proxies for internal routing logic.
```python
import requests
from app.core.config import settings

http_session = requests.Session()
http_session.trust_env = False  # Ignore system proxy settings
qdrant_base_url = f"http://{settings.QDRANT_HOST}:{settings.QDRANT_PORT}"
collection_name = settings.QDRANT_COLLECTION_NAME
```

### Checking and Creating Collection
Before inserting points, ensure the collection exists. If it returns 404, we create it.
```python
rest_url = f"{qdrant_base_url}/collections/{collection_name}"
resp = http_session.get(rest_url, timeout=15)
if resp.status_code == 404:
    create_payload = {
        "vectors": {
            "size": settings.EMBEDDING_DIMENSION, 
            "distance": "Cosine"
        }
    }
    http_session.put(rest_url, json=create_payload, timeout=30)
```

### Upserting Embeddings (Storage)
Format points into batches and `PUT` to the `/points` endpoint. Include rich metadata via `payload`.
```python
import uuid

# Points preparation
points = []
for i, emb in enumerate(all_embeddings):
    points.append({
        "id": str(uuid.uuid4()),
        "vector": emb,
        "payload": {
            "file_id": str(file_id),
            "project_id": str(project_id),
            "text": chunks[i],
            "chunk_index": i
        }
    })

# Batch Upserting
upsert_url = f"{qdrant_base_url}/collections/{collection_name}/points"
for i in range(0, len(points), 100):
    batch = points[i:i + 100]
    upsert_payload = {"points": batch}
    http_session.put(upsert_url, json=upsert_payload, timeout=30)
```

### Deleting Vectors by Metadata Filter
Remove stale embeddings using the `/points/delete` endpoint with a filter.
```python
delete_url = f"{qdrant_base_url}/collections/{collection_name}/points/delete"
delete_payload = {
    "filter": {
        "must": [{"key": "file_id", "match": {"value": str(file_id)}}]
    }
}
http_session.post(delete_url, json=delete_payload, timeout=30)
```

### Searching Vectors (Retrieval)
Use the `/points/search` POST endpoint.
```python
search_url = f"{qdrant_base_url}/collections/{collection_name}/points/search"
search_payload = {
    "vector": query_embedding,
    "limit": 5,
    "with_payload": True,
    # Optional filtering
    "filter": {
        "must": [{"key": "project_id", "match": {"value": str(project_id)}}]
    }
}
resp = http_session.post(search_url, json=search_payload, timeout=30)
results = resp.json().get("result", [])
```
